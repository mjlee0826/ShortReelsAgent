# 簡報：技術架構三張圖說明

> 投資人 pitch 的「技術障礙」段(約佔 10 分鐘的 1/3、3 頁、每頁約 1 分鐘)。
> 鐵律:**每張先講標題那句結論,圖只是佐證**;不要「來,我解釋一下這張圖」。
> 飛輪 / few-shot 不另開頁——在第 1 頁講一句、第 3 頁當 punchline、結尾頁當未來護城河主秀。

---

## 圖 1：總架構圖（establishing shot / 地圖）

**標題（結論句）**：「我們的護城河在這兩塊綠——租來的大腦只是中間一站。」

**畫面**：由左到右的 5 節點水平流程
`Raw Media → Deep Understanding → Director (LLM) → Editable Blueprint → Render`

- 顏色語意：綠 = 我們的護城河（第 2、4 格）、灰 = 租用（第 3 格 Director LLM）、中性（第 1、5 格）。
- 圖例：綠 = `Our Moat`、灰 = `Rented`。
- 兩條回饋箭頭：
  - **青綠**：`Render(5) → Editable Blueprint(4)`，標 `Edit & Refine with AI`（產品迴圈：看 render → 回去改藍圖）。
  - **橘色**：`Editable Blueprint(4) → Director (LLM)(3)`，標 `User Edits → Preference Data`（few-shot 飛輪：使用者編輯變成偏好範例，餵進導演的 prompt；箭頭**接進** Director）。

**功能**：地圖。一眼看出護城河落在哪、哪塊是租來的——這正是和「套殼隊伍」拉開差距的點。
**講法**：帶過五格流程，點名「綠的是我們的、灰的是租的」，並用一句帶出橘箭頭（每次編輯回饋成範例餵導演）。深度留 Q&A。

---

## 圖 2：DAG + 資源分配（成本護城河）

**標題（結論句）**：「每支成本 ≈ 電費。」

**畫面**：主視覺是 before/after 吞吐甘特圖
- 循序版：`[decode][分析…][whisper][場景]` ← 一堆空格、資源輪流閒置。
- 我們的：各 Stage 疊在一起、卡片排滿 ← DAG + 四資源池（IO/CPU/GPU/API）讓 GPU/CPU/網路「同時上工」，吞吐 ×N。
- 壓軸一行：**GPU 顯存預算調度（BudgetGate）= 榨到滿還不崩**，是在真實共用 GPU 上用一次次當機換來的 know-how，抄不走。

**功能**：成本護城河（單位經濟）。DAG 是「機制」、不是主角;主角是「每支成本≈電費」。
**講法**：「別人租算力做 AI 剪輯;我們把同一張卡榨到滿、榨滿還不爆,所以每支更便宜更快。」深層依賴圖 / 顯存演算法留 Q&A,用來回擊「你們是不是只是串 API」。
**競賽前要量化**：循序 vs 並行的處理時間差幾倍、每支成本（我們 vs 純雲端）。

---

## 圖 3：Editable Blueprint（產品護城河 + 飛輪 punchline）

**標題（結論句）**：「交付的是時間軸,不是影片。」

**畫面**：split-screen 對比
- 別家：`[ 黑盒影片 ]` — 不滿意只能整支重生。
- 我們：`[ 可編輯時間軸 ]` — 拖字幕 / 換調色 / 換配樂,改完即時預覽。

**Punchline（接飛輪,讓第 3 頁從「功能」升級成「資料護城河」）**：
> 「正因為輸出是**可編輯藍圖**,每一次編輯才變成資料 → **今天當 few-shot 餵導演、明天訓練我們自己的導演模型**。競品的 MP4 收不到。」

**功能**：產品護城河（掌控感 → 留存）+ 資料飛輪的種子。
**誠實時態**：捕捉 + few-shot = **現在式**(已上線);訓練自有模型 = **未來式**(需要量,且只能微調自架開源模型,不能 fine-tune Claude)。資料量問到就答「機制已閉合、量隨使用者長」。

---

## 不要做
- **不加第 4 頁講 few-shot**——它已在圖 1、會在圖 3 當 punchline、在結尾頁當主秀;另開頁會模糊兩個工程 moat、爆掉 1/3 時間預算,並招來「給我看數據」的逼問。
- 每張圖都**先講結論、再讓圖佐證**;不要逐格解說架構。
